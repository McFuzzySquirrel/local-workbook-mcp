# Key Learnings

- Pivot tool MCP parameters are snake_case (`pivot_table`, `include_filters`, `max_rows`) — they mirror C# method parameter names; old docs had camelCase which would break external MCP client integrations silently
- The `python` command is not available on this system; must use `python3`
- All projects confirmed on `net10.0` via grep across .csproj files — always verify programmatically rather than trusting existing docs
- The existing ejs-session-2026-03-12-01 was created in an earlier session today (manual test scripts); daily sequence numbering requires checking the directory before initializing
- README `appsettings.json` code blocks can easily drift from the actual config structure; cross-check against the real file
- When `git add -A` is used in a commit and README edits were already in the working tree, they get bundled into the same commit — no need for a separate push
